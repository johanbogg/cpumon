# CpuMon QA pass - 2026-05-21

## Applied in this pass

- Hardened one-shot web install links:
  - Failed bundle downloads no longer consume the install link before a staged release/artifact is available.
  - The generated admin `install.bat` now quotes server/token/thumbprint arguments.
  - Install-link server address input rejects shell metacharacters.
- Bounded `/ws/log` buffering so a stalled browser cannot grow an unbounded in-memory queue.
- Added idle cleanup for web terminal sessions and closes stale remote shells after 15 minutes of inactivity.
- Routed web file-upload command results with the file browser session id so remote upload success/failure can reach the web session.
- Added a fallback temp staging path for web file downloads if `ProgramData\CpuMon\webdl` cannot be created, which also makes tests reliable in restricted environments.
- Marked the Windows service agent pipe disconnected when direct writes fail, so the service can recover instead of silently dropping agent-targeted commands.
- Added file-transfer integrity checks for direct and PAW downloads by validating incoming chunk offsets.
- Avoided queuing direct file-upload chunks after a disconnect; uploads now fail visibly instead of reporting success after partial transfer.
- Cleaned direct file-browser download state/temp files when the dialog closes.
- Improved Linux stop/restart behavior:
  - SIGTERM/SIGINT now closes the socket and wakes the client loop.
  - Self restart/stop uses `systemd-run` when available so the helper is less likely to die in the service cgroup.
- Hardened Debian/RPM Linux installer config writes with shell-safe quoting.
- Corrected README Linux config variable names to `CPUMON_SERVER_IP` and `CPUMON_TOKEN`.

## Worth fixing later

- `FileBrowserDialog`, `TerminalDialog`, and related WinForms file/terminal code are very dense single-line constructors/methods. They work, but future fixes are slower and riskier than necessary. A readability-only split would pay off.
- PAW upload dispatch uses a fire-and-forget delegate, so it cannot detect a broken relay path as cleanly as direct uploads now can.
- Linux installer `read_existing_config` reads raw assignment text rather than evaluating shell quoting. It is safe, but prompts may show escaped values if a token/server value contains special characters.
- RDP still has protocol/session ownership debt: input/frame ownership and backpressure would be better isolated from UI dialogs before any major RDP quality work.
- The web file browser currently polls session state. A future websocket-backed stream could reduce latency and repeated HTTP traffic, but the current polling path is simpler and acceptable for home use.
